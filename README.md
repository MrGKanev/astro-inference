# astro-inference

Astro integration that automatically generates LLM-friendly pages for your site.

Adds two routes to any Astro project:

- `/llms.txt` - structured plain-text index of your site, following the [llms.txt standard](https://llmstxt.org/)
- `/[page]/machine.txt` - clean `text/plain` markdown version of any page, stripped of navigation, scripts, and visual noise

## Installation

```bash
npm install astro-inference
```

## Setup

```ts
// astro.config.mjs
import { defineConfig } from 'astro/config';
import astroInference from 'astro-inference';

export default defineConfig({
  integrations: [
    astroInference({
      siteName: 'My Site',
      siteDescription: 'What my site does in one sentence.',
    }),
  ],
});
```

That's it. No other configuration required.

## Routes

### `/llms.txt`

A machine-readable index of your entire site. Grouped by content collection, with titles and descriptions pulled from frontmatter.

```
# My Site
> What my site does in one sentence.

## Blog

- [Getting started](https://mysite.com/blog/getting-started) - A quick intro
- [Another post](https://mysite.com/blog/another-post)

## Machine-readable pages

Each page on this site has a plain-text version at `[page-url]/machine.txt`.
Example: `https://mysite.com/about/machine.txt`
```

### `/[page]/machine.txt`

A plain-text markdown version of any page on your site. Strips nav, footer, scripts, and styles - leaving only the semantic content.

```
# My Site
> Source: https://mysite.com/about
> Machine-readable version - 2026-04-07

---

# About Us

We build great things for great people.

## Our values

- Honesty
- Speed
- Quality

---
Generated by astro-inference | https://mysite.com/llms.txt
```

Works for any route: `/about/machine.txt`, `/blog/my-post/machine.txt`, `/docs/guide/machine.txt`.

## Options

| Option | Type | Default | Description |
|---|---|---|---|
| `siteName` | `string` | hostname | Heading used in `llms.txt` |
| `siteDescription` | `string` | - | One-line tagline shown below the heading |
| `exclude` | `string[]` | `[]` | Paths to exclude. Supports `/*` wildcard suffix. |
| `llmsTxtPath` | `string` | `/llms.txt` | Where `llms.txt` is served |
| `machineSuffix` | `string` | `machine.txt` | Suffix appended to each page route |

### Example with all options

```ts
astroInference({
  siteName: 'Acme Corp',
  siteDescription: 'We make the best widgets on the internet.',
  exclude: ['/admin/*', '/private', '/thank-you'],
  llmsTxtPath: '/llms.txt',
  machineSuffix: 'machine.txt',
})
```

## Content Collections

If your project uses [Astro Content Collections](https://docs.astro.build/en/guides/content-collections/), `astro-inference` picks them up automatically.

Supported collection names: `blog`, `posts`, `articles`, `pages`, `docs`, `work`, `projects`, `portfolio`, `news`, `events`.

Frontmatter fields used:

| Field | Used in |
|---|---|
| `title` | Link label in `llms.txt` |
| `description` or `summary` | Description after `-` in `llms.txt` |

## How it works

1. On `astro:config:setup`, two routes are injected into your Astro app via `injectRoute()`.
2. `/llms.txt` reads your content collections at request time and builds a grouped markdown index.
3. `/[...slug]/machine.txt` fetches the original page HTML, strips non-semantic elements (nav, footer, scripts, styles), and converts the remainder to clean markdown.

No build-time scanning, no file system crawling - everything runs through Astro's existing request pipeline.

## Requirements

- Astro 4.0 or later
- Node.js 18 or later
